- 日期：2026-07-17
- 分类：tools/自动化
- 蒸馏日期：2026-07-18

## 痛点
1. 每日重复任务缺乏系统化流程
2. 选题容易撞车，昨天写过什么记不清
3. 发布后检查靠手感，没有固定标准
4. 失败原因没有沉淀，同样的错误反复犯
5. 自动化任务只写了动作，没写清最终结果
6. 自动化为了完成任务，把不完整结果也算完成

## 解法
1. 自动化任务先写结果，再写动作——明确可验收的结果
2. 把任务边界写进项目规则（AGENTS.md）
3. 配记忆文件：上次运行时间、上次发布内容、本次避开的主题、发现的问题
4. 选题先查冲突：提候选词→查标题/slug/摘要→只保留无冲突的
5. 写作提示包含审稿要求：真实场景开头、可执行动作、避免万能开场
6. 特色图流程化：强相关主题图、1200×630、alt描述主题
7. 发布后检查脚本输出JSON：HTTP状态、title、meta、keywords、alt、H1、内链
8. 失败处理写进任务：发布失败保留草稿、图片失败换图、选题冲突换候选、凭据缺失停止

## 可执行规则
1. 自动化任务先写可验收的结果描述，再写执行步骤
2. 每次任务配记忆文件，记录上次运行状态
3. 发布前必须查冲突，不硬改标题绕过去
4. 失败处理必须写进任务，区分"完成""失败""需人工决策"
