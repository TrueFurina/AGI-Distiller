# 自动化任务编写器

用户请求: $ARGUMENTS

## 规则

1. 自动化任务先写可验收的结果描述，再写执行步骤
2. 每次任务配记忆文件，记录上次运行状态
3. 发布前必须查冲突，不硬改标题绕过去
4. 失败处理必须写进任务，区分"完成""失败""需人工决策"

---

## 步骤 1：确定任务结果

先写清楚"做到什么算完成"，再写动作。

```
## 任务结果
- 每次运行产出：
- 数量：
- 验收标准：
- 检查项：
```

## 步骤 2：定义边界和规则

```
## 边界
- 可以做：
- 不可以做：
- 凭据来源：
- 失败处理：
  - 发布失败：
  - 图片失败：
  - 选题冲突：
  - 凭据缺失：
```

## 步骤 3：配记忆文件

```
## 记忆文件格式
- 上次运行时间：
- 上次产出（标题/链接/关键词）：
- 本次避开的主题：
- 上次发现的问题：
- 下次要注意的点：
```

## 步骤 4：写执行流程

```
## 执行流程
1. 读记忆文件，确认上次状态
2. 选题/提候选词
3. 查冲突（搜索标题/slug/摘要）
4. 执行核心任务
5. 发布后检查（HTTP状态/title/meta/keywords/alt/H1/内链）
6. 更新记忆文件
7. 输出结果汇总
```

## 输出格式

```
## 自动化任务说明书

### 任务名
[任务名称]

### 结果描述
[做到什么算完成]

### 边界
[可以做/不可以做/失败处理]

### 记忆文件
[存储位置和格式]

### 执行流程
[步骤1 → 步骤2 → ...]

### 验收清单
[检查项列表]
```
